## 核心任务完成
成功完成了 AI Agent 页面中 `Market Scanner` 模块的全面审计与优化，重点解决了五个核心问题：

### 1. 数据真实性核查 ✅
**发现的问题**:
- 原前端 `getStockNews()`、`analyzeTrend()`、`generateScannerReason()` 函数使用随机/模拟数据
- 新闻情绪、事件风险、趋势分数均为随机生成
- Scanner Reason 从模板句子随机选择

**修复方案**:
- 建立真实数据链路：前端 → `/api/ai/market/scanner` → 真实数据源 → DeepSeek
- 移除了所有随机数据生成逻辑

### 2. 公司名显示添加 ✅
**实现**:
- 在 Market Scanner 表格的 Symbol 列下方添加公司名称显示
- 数据来源：Finnhub profile API (`/stock/profile2`)
- 后备机制：仅当 API 失败时使用硬编码映射 (AAPL → Apple Inc. 等)
- 前端渲染：在 symbol 按钮下显示小字体公司名

### 3. 新闻数据真实化 ✅
**新闻数据链路**:
```
前端 → 后端 API → fetch_finnhub_company_news() → Finnhub /company-news → 分析 → 前端显示
```

**新闻分析逻辑**:
- 获取过去7天真实新闻 (headlines + summaries)
- 使用 Finnhub 提供的 `sentiment_score` (-1 到 1) 计算平均情感
- 基于关键词分析事件风险：
  - 高风险：'earnings miss', 'guidance cut', 'lawsuit', 'investigation'
  - 中风险：'earnings', 'guidance', 'analyst actions'
- 识别主要催化剂 (earnings, guidance, analyst actions 等)

### 4. DeepSeek 真正参与判断 ✅
**DeepSeek 集成点**:
- `analyze_trend_with_deepseek()`: 核心趋势分析函数
- **输入**: 真实市场数据 + 新闻分析结果
- **输出**: `trendLabel`, `trendScore`, `trendConfidence`, `scannerReason`
- **提示词**: "作为专业的量化分析师，请分析以下股票并给出趋势判断..."

**DeepSeek 处理的数据**:
- 价格、变动%、成交量 (真实 Alpaca/Finnhub 数据)
- 新闻情绪、事件风险、主要催化剂 (真实新闻分析)
- 公司名称、板块信息

**失败处理**:
- 如果 DeepSeek API 密钥无效或调用失败，降级到 `analyze_trend_locally()`
- 明确标记本地分析，不伪造 AI 总结

### 5. UI 专业化优化 ✅
**Summary 区域优化**:
- 使用 Ant Design `Statistic` 组件显示关键指标
- 统一颜色编码和专业图标
- 添加 "Last Scan Time" 显示

**表格优化**:
1. **Symbol 列**: 添加公司名显示，限制宽度，溢出处理

3. **Score 列**: 趋势分数 + 置信度百分比

5. **Volume 列**: 格式化为百万单位 (45.7M)
6. **News 列**: 情绪标签 + 事件风险

**过滤排序控制**:
