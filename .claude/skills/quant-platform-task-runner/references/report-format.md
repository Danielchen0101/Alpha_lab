# Report Format

Use this compact report format after every task.

## Required format

```
Files checked:
...

Files changed:
...

Root cause:
...

Changes made:
...

Verification:
  Command:
  Result:

Before / after:
  Before: only the changed block
  After:  only the changed block

Remaining risk:
...
```

## Rules

- Do not paste entire large files.
- Do not include unrelated code.
- Do not claim success if build/test failed.
- If something was skipped, say exactly why.
- If a tool/skill is unavailable, say it clearly.
